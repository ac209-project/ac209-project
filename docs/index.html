Problem Statement & Motivation

The purpose of this project is to use publicly available data through Spotify's API, the Spotipy python library, and audio features in the Million Song Dataset (MSD) to predict the success of Spotify user playlists, and create a tool to build successful playlists based on user specified features (genre, artist, etc.).

Throughout this project "playlist success" (our response variable) is defined as the number of playlist followers. We have employed machine-learning regression techniques taught throughout the semester as well as Word2vec modeling to predict playlist success.

Introduction & Description of Data

Baseline Data:
Baseline data was collected primarily from two sources - the Spotify Web API and the MSD. After sourcing the Spotify users who “own” (the playlists are attached to their accounts) a large number of playlists, the spotipy library was employed to collect user playlists, tracks, user followers, and genre data from the API, each saved as JSON files. Genre information for each playlist was harder to source and incorporate than one would think. It does not come directly from each playlist, but rather from the artists of each track as a separate query. We also collected celebrity names from US Magazine to determine if usernames corresponded to recognized celebrities. Before combining into one comprehensive DataFrame, the separate files have the following basic form:

Word2vec on Playlist Names:
In order to go beyond the scope of 109A and incorporate a method not discussed in class, we decided to apply a word-to-vec (w2v) transform to as many applicable words as there are in the set of playlist names. The core intuition being that the playlist name is what represents the playlist to the outside world. A bad playlist name can keep people from following or listening to it, even if the tracks within are great.

Click here to explore the w2v model training and implentation specific to our project.

Techique:
By using word-to-vec we seek to cluster the playlist names into meaningful categories based on the sum of the vectors of known words in each playlist. This implicitly seperates playlists with longer (presumably more informative) names by giving them a total higher magnitude in the 100-dimensional space of our w2v transform, but seems to be a good aggregation technique.

Difficulties:
Ultimately, there will be some difficulties with misspellings, different languages, and internet slang, and we will doubtless lose information in this quick application of w2v, but we think the output will still be a meaningful feature in our final model.
